# チーム運用方針

## ドキュメント管理

| フェーズ | ツール | 用途 |
|---|---|---|
| 議論・ブレスト中 | Notion | 議題共有、意見出し、コメント |
| 決定後 | md（Git管理） | 仕様の記録、Claude参照用 |

Notionは消耗品、mdは記録。同じ内容を二重管理しない。

### フロー

```
Notionで議論して決まったら
  ↓
docs/ 配下のmdに書き起こす
  ↓
git push
```

## 連絡・共有

- 議題・共有事項：Notionで作成 → LINEで共有

## ローカル編集環境

- mdファイルの編集にはObsidianを推奨（`docs/`をVaultとして開く）
